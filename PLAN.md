# Prompter — Live Script Guidance for High-Stakes Conversations

> Your words, perfectly delivered. Every time.

## Problem

Pharmacists, sales professionals, and consultants deliver complex, high-stakes scripts to patients and clients. These scripts contain critical safety disclosures, conditional branches, and interactive pause points. Today they either:

1. **Memorize** — unreliable, skips critical sections, high cognitive load
2. **Read from paper/screen** — breaks eye contact, sounds robotic, can't track position after ad-libs
3. **Use generic teleprompters** — dumb scroll, no pause detection, no branching, no compliance tracking

The result: missed disclosures, inconsistent delivery, compliance risk, and worse outcomes for patients/clients.

## Solution

**Prompter** — a desktop teleprompter that understands conversation. It tracks the speaker's voice against a known script in real-time, scrolling intelligently, pausing when the other person speaks or the user ad-libs, and handling conditional branches with a tap. After the session, it produces a compliance report showing exactly what was covered.

## Core Use Cases

### 1. Pharmacy MTM Consultations (Primary — RxVIP)
- Pharmacist delivers a 15-20 minute medication therapy management consultation
- Script includes safety-critical drug interaction disclosures
- Patient asks questions, pharmacist ad-libs explanations
- Compliance requirement: all safety points must be covered
- Script is generated by the RxVIP pharmacy ecosystem (Next.js app)

### 2. Sales Calls
- Sales rep follows a structured pitch with objection handling branches
- Prospect speaks ~40-50% of the time
- Rep needs to sound natural, not scripted
- Manager wants to know which sections were covered and skipped

### 3. Presentations / Video Recording (Future)
- Speaker reads from a teleprompter while looking at camera
- No "other speaker" — simpler mode
- Word-level tracking for smooth delivery

## Architecture

### Separate app, shared engine

Prompter is a standalone Tauri v2 desktop app that uses `minutes-core` as a Rust library dependency for audio capture, whisper transcription, and speaker diarization. This keeps Minutes lean (passive listener) while Prompter is purpose-built (active guide).

```
┌─────────────────────────────────┐     ┌──────────────────────────┐
│  Script Source                  │     │  Prompter (Tauri v2)     │
│  (RxVIP app, file, clipboard)  │     │                          │
│                                 │     │  Script Parser           │
│  saves .script.md ─────────────┼──►  │  Alignment Engine        │
│  or clipboard paste             │     │  Teleprompter UI         │
│                                 │     │  Compliance Tracker      │
└─────────────────────────────────┘     │                          │
                                        │  ┌────────────────────┐  │
                                        │  │ minutes-core (lib) │  │
                                        │  │ audio, whisper,    │  │
                                        │  │ diarization        │  │
                                        │  └────────────────────┘  │
                                        │           │              │
                                        │           ▼              │
                                        │  ~/meetings/consults/    │
                                        │  (Minutes indexes these) │
                                        └──────────────────────────┘
```

### minutes-core as a library

This requires refactoring minutes-core to:
- Expose a clean public API for audio capture, streaming whisper, and diarization
- Add feature flags so consumers can opt into only what they need
- Keep the internal pipeline (summarize, search, notes, watch) behind feature gates
- Path dependency during development, crate publish later if others want it

### Script Format (.script.md)

Annotated markdown that the parser understands:

```markdown
---
title: MTM Consultation — {{patient_name}}
type: pharmacy-consultation
variables:
  patient_name: Jane Smith
  medications: [Warfarin, Metformin]
  supplements: [Garlic, Ginkgo, St. John's Wort]
  symptoms: [dizziness, bruising]
  insurance: Humana
estimated_duration: 18min
---

# Intro

Hi, thanks for meeting with me today. My goal is simple...

> PAUSE: Does that sound helpful to you?

# Explanation of Findings

Let me walk you through exactly what I see happening...

> PAUSE: Does this make sense so far?

> PAUSE: Have you noticed tingling in your feet or memory loss?

# Recommendations

Based on everything we just uncovered...

> BRANCH: Would you like me to get this plan organized?
>> YES
Is there anyone else—a spouse, adult child...
>> NO
May I have your permission to share...

# Closing

Before we wrap up—do you have any questions?
```

### Tracking Engine

Three tiers of intelligence, built incrementally:

**Tier 1 — VAD-guided scroll (MVP)**
- Voice Activity Detection: scroll when user speaks, pause when silent
- VAD-only for v1 (no pyannote diarization — removes Python dependency entirely)
- Manual speed adjustment
- Tap to pause/resume, tap section headers to jump
- Audio chunking: split on silence boundaries (VAD-based), not fixed time windows

**Tier 2 — Speech-guided tracking**
- Stream 3-second audio chunks through whisper
- Fuzzy-match recognized text against ±200 word window around current position
- Advance scroll to matched position
- When match confidence drops + VAD active → ad-lib or patient speaking → pause
- Auto-pause at marked pause points

**Tier 3 — Word-level highlighting (future)**
- Forced alignment for word-granularity tracking
- Current word highlighted, read words dimmed
- Professional teleprompter feel

### Compliance Output

After session, write a report to the meeting file frontmatter:

```yaml
compliance:
  sections_covered: [Intro, Explanation, Recommendations, Closing]
  sections_skipped: []
  duration: 17:42
  section_times:
    Intro: 2:15
    Explanation: 6:30
    Recommendations: 7:12
    Closing: 1:45
  pause_points_reached: 4/4
  branches_taken:
    "Would you like me to get this plan organized?": "YES"
  script_adherence: 94%
```

**Interaction states:**
| Feature | Loading | Empty | Error | Success | Partial |
|---------|---------|-------|-------|---------|---------|
| Script load | Spinner + "Loading..." | Drop zone + "Drop .script.md or paste" + recent scripts | "Parse error at line 12" + offending line | Script in teleprompter | N/A |
| Audio session | "Connecting mic..." + pulse | N/A | "No mic found. Check System Settings" | Green dot + timer | "Mic disconnected" + pause |
| Branch point | N/A | N/A | N/A | Two large tap targets | Pulsing nudge after 30s |
| Session end | "Saving..." | N/A | "Saved to temp dir" + recovery | Warm completion: checkmarks per section, total time | "Partial report saved" |
| First run | N/A | "Download whisper model" + button + size | "Download failed" | "Model ready. Load a script." | N/A |

**Emotional design:** The teleprompter should feel like a confident coach, not a surveillance system. Session completion shows warm encouragement (checkmarks, "Great consultation"). Detailed compliance YAML is saved to disk for managers.

**Scroll behavior:** Smooth sentence-by-sentence advancement. When the current sentence advances, the previous one fades to 20% opacity over 300ms. No jarring jumps.

**Keyboard shortcuts (essential for hands-on-keyboard consultations):**
| Key | Action |
|-----|--------|
| Space | Pause / Resume |
| Escape | End session (with confirmation) |
| Up/Down | Manual scroll |
| 1-4 | Jump to section by number |
| Tab | Cycle branch options |
| Enter | Select highlighted branch option |
| +/- | Increase / decrease font size |

## What minutes-core needs

To serve as a library, minutes-core needs:

1. **Feature flags** in Cargo.toml:
   - `capture` — audio capture via cpal
   - `transcribe` — whisper.cpp integration
   - `diarize` — pyannote speaker detection
   - `full` — everything (current default for Minutes CLI)
   - `streaming` — new: real-time whisper with chunk-based processing

2. **Clean public API**:
   - `AudioStream` — start/stop capture, get audio chunks
   - `Transcriber` — init with model, transcribe chunk → text
   - `Diarizer` — init, process audio → speaker segments
   - Keep pipeline, summarize, search, watch, notes behind `full` feature

3. **Streaming transcription** (new capability):
   - Currently whisper processes complete files
   - Need: process 3-second audio chunks in real-time
   - Return partial results with word timestamps
   - This benefits Minutes too (live transcription preview)

## Review Decisions (CEO Review 2026-03-20)

### Accepted scope additions
- SPEC.md documenting .script.md as an open protocol
- Script versioning: `version` field in frontmatter, tracked in compliance output
- Full transcript in Minutes-compatible markdown alongside compliance YAML

### Architecture decisions
- **VAD-based audio chunking** — split on silence boundaries, not fixed 3-second windows
- **minutes-core facade** — expose `pub mod api` module, not raw internal modules
- **VAD-only for v1** — no pyannote/diarization, removes Python dependency entirely
- **Error handling** — OOM: cap chunk size + retry. Write failures: temp dir fallback + alert
- **PII cleanup** — move .script.md to processed/ after session, patient ID in compliance YAML
- **Branch timeout** — gentle nudge after 30s if pharmacist doesn't tap
- **Graceful shutdown** — window close triggers same flow as Stop button, saves partial data
- **UI states** — empty state (drop script + recent list), error state (parse failure + line), first-run (model download)
